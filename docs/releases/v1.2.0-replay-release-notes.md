# v1.2.0 Replay Release Notes

Date: 2026-04-28

## Goal

Add a human-readable history replay layer on top of the existing Claude Code and Codex JSONL scanners. The release focuses on reviewing past sessions as chat records while keeping raw JSONL available for debugging.

## Implementation Summary

- Added a replay reader in `electron/replay.ts`.
- Added IPC bridge method `getReplaySession(sessionId, source, options)`.
- Added a standalone `ReplayPage` and a sidebar entry between Sessions and Models.
- Added replay tabs inside session details:
  - `Overview`
  - `Replay`
  - `Raw events`
- Added `ConversationView` for chat-style rendering.
- Added Markdown image, image attachment, table, and KaTeX formula rendering.
- Added immersive fullscreen replay mode that hides app navigation and session lists.

## Replay Rules

- `conversationOnly: true` returns only user and assistant messages.
- Technical setup text such as environment, permissions, app context, plugin instructions, and AGENTS instructions is hidden from replay.
- Tool calls, tool results, token usage rows, and metadata stay visible in Raw events.
- For assistant messages, multiple staged replies in one user turn are collapsed to the final assistant message.
- Raw JSON rows are only attached when `includeRaw: true`.

## Data Source Behavior

- Claude Code replay reads candidate `.jsonl` files under `~/.claude/projects`.
- Codex replay reads candidate `.jsonl` files under `~/.codex/sessions`.
- When session records already identify the source file, replay loading is scoped to those candidate files to avoid scanning unrelated local or remote cache files.
- Remote SSH cache replay uses the same candidate-file safety rules.

## UI Notes

- The small replay preview remains available from the Sessions detail pane.
- The full Replay page is better for long sessions.
- The fullscreen button opens an overlay with only the conversation, search box, and exit control.
- `Esc` exits fullscreen replay.
